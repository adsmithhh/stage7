# stage7

An orchestrated simulation platform built on API-mediated package architecture. Zones, NPCs, emotional fields, doctrine systems, and real-time rendering — all running through clean API boundaries.

Part of the [insideworld](https://github.com/adsmithhh/insideworld) lineage.

## Architecture

```
jj.py                          ← entrypoint
├── runtime/                   ← application lifecycle and orchestration
├── simulation/                ← world engine, NPCs, doctrine
├── rendering/                 ← visual projection and UI (pygame / OpenGL)
├── emx/                       ← emotional field engine
├── persistence/               ← save/load and state continuity
├── tuning/                    ← runtime parameter calibration
├── config/                    ← static schemas, constants, overrides
└── diagnostics/               ← architectural and behavioral validation
```

Each package exposes a single `api.py` surface. No cross-package imports bypass this boundary.

## Systems

- **Simulation** — multi-zone world with NPC agents, commitment planning, zone-locking, work budgets, and a pantheon contribution model
- **EMX** — emotional matrix engine with cell grids, weather systems, dependency coupling, personality baselines, overuse backlash, and emotional complexes
- **Rendering** — dual-backend (software + OpenGL) real-time visualisation with inspector panels, EMX overlays, and NPC state export
- **Doctrine** — behavioral governance layer influencing NPC decision-making
- **Persistence** — slot-based save/load with NPC state snapshots
- **Tuning** — live parameter calibration during runtime
- **Diagnostics** — baseline validation and architecture auditing
